# Linux

## 1. Ubuntu와 터미널 감 잡기

### 1.1 터미널(Terminal)
- 마우스 클릭 대신 텍스트 명령으로 시스템을 제어하는 인터페이스

### 1.2 기본 명령어

| 명령어 | 설명 | 사용법 |
|--------|------|--------|
| `pwd` | 현재 작업 디렉토리 경로 출력 | `pwd` |
| `ls` | 현재 디렉토리의 파일·폴더 목록 출력 | `ls` |
| `cd` | 디렉토리 이동 | `cd 경로` |

**`ls` 옵션**
- `ls -l` — 자세히 보기
- `ls -a` — 숨김 파일 포함

**`cd` 단축 경로**
- `cd ..` — 상위 디렉토리
- `cd ~` 또는 `cd` — 홈 디렉토리
- `cd /` — 루트 디렉토리

### 1.3 경로
- **절대 경로**: 루트 디렉토리(/)부터 시작하는 경로
- **상대 경로**: 현재 디렉토리부터 시작하는 경로

### 1.4 주의할 점
- **대소문자 구분**: Linux는 대소문자를 구분함
- **공백**: 파일명이나 디렉토리명에 공백이 있으면 `" "`로 감싸줘야 함

## 2. 파일·디렉토리 다루기

| 명령어 | 설명 | 사용법 |
|--------|------|--------|
| `mkdir` | 디렉토리 생성 | `mkdir 폴더이름` |
| `touch` | 빈 파일 생성 | `touch 파일이름` |
| `cp` | 파일 복사 | `cp 원본 대상` |
| `mv` | 파일 이동 / 이름 변경 | `mv 파일 폴더` / `mv 원본 새이름` |
| `rm` | 파일 삭제 | `rm 파일이름` |

**옵션 정리**
- `mkdir -p` — 계층 구조의 상위 디렉토리까지 한 번에 생성
- `cp -r` — 디렉토리 통째로 복사
- `rm -r` — 디렉토리 삭제
- `rm -f` — 강제 삭제

> [!WARNING]
> `rm`은 복구 불가능하며 확인 없이 즉시 삭제됩니다. 휴지통이 없습니다.

## 3. 파일 내용 보기·편집

### 내용 보기

| 명령어 | 설명 | 사용법 |
|--------|------|--------|
| `cat` | 파일 전체 내용을 터미널에 출력 | `cat 파일이름` |
| `less` | 별도 뷰어에서 파일 내용 출력 | `less 파일이름` |
| `head` | 파일 앞부분 출력 (기본 10줄) | `head 파일이름`, `head -n 숫자` |
| `tail` | 파일 뒷부분 출력 (기본 10줄) | `tail 파일이름`, `tail -n 숫자` |

**`less` 단축키**
- `q` — 종료 / 방향키 — 스크롤 / `b` — 이전 페이지 / `Space` — 다음 페이지

**`tail` 옵션**
- `tail -f` — 파일 내용 실시간 출력 (`Ctrl+C`로 종료)

> [!WARNING]
> `cat`은 파일이 길면 화면이 한 번에 지나갑니다. 긴 파일은 `less` 사용을 권장합니다.

### 내용 편집

**`nano 파일이름`**
- `Ctrl+O` → `Enter` — 저장
- `Ctrl+X` — 종료

## 환경 설정

**`source`**
- 현재 셸에 스크립트를 적용하는 명령어 (터미널은 새로 열 때마다 환경이 초기화됨)

```bash
source /opt/ros/humble/setup.bash   # ROS 2 Humble 환경을 현재 터미널에 활성화
```
> PATH 추가 (ros2 명령어 사용 가능) / ROS 패키지 경로 등록 / 라이브러리 경로 설정 / 각종 ROS 환경변수 세팅

## 4. 검색과 찾기

| 명령어 | 설명 | 사용법 |
|--------|------|--------|
| `find` | 파일 검색 | `find 경로 -name "파일이름"` |
| `grep` | 텍스트 검색 | `grep "검색어" 파일이름` |
| `which` | 실행 파일 경로 확인 | `which 명령어` |

**`grep` 옵션**
- `-i` — 대소문자 무시
- `-r "검색어" 디렉토리` — 디렉토리 전체 검색
- `-n` — 줄 번호 표시

**`find` + `grep` 조합**
```bash
find 경로 -name "*.py" | xargs grep "검색어"
```

## 6. 패키지 설치·관리

### 6.1 개념
- **저장소** — 패키지가 저장된 원격 서버
- **패키지** — 설치 가능한 프로그램 단위

### 6.2 apt 명령어

| 명령어 | 설명 |
|--------|------|
| `sudo apt update` | 최신 패키지 목록 가져오기 |
| `sudo apt upgrade` | 설치된 패키지를 최신 버전으로 업그레이드 |
| `sudo apt install 패키지이름` | 패키지 설치 |
| `sudo apt remove 패키지이름` | 패키지 삭제 |
| `sudo apt purge 패키지이름` | 패키지 + 설정 파일까지 삭제 |
| `apt search 키워드` | 패키지 검색 |
| `apt list --installed` | 설치된 패키지 목록 조회 |

> [!WARNING]
> `apt install` 전에 반드시 `apt update`를 먼저 실행하세요.

특정 패키지 설치 여부 확인:
```bash
apt list --installed | grep 패키지이름
```

## 7. 프로세스·시스템 확인

### 7.1 개념
- **프로세스** — 현재 실행 중인 프로그램
- **PID** — 프로세스 고유 식별 번호

### 7.2 프로세스 명령어

| 명령어 | 설명 | 사용법 |
|--------|------|--------|
| `ps` | 현재 터미널의 프로세스 목록 | `ps`, `ps aux` |
| `top` | 실시간 프로세스 모니터링 | `top` (종료: `q`) |
| `kill` | 프로세스 종료 | `kill PID`, `kill -9 PID` |

- `ps aux` — 시스템 전체 프로세스 조회
- `kill -9` — 강제 종료 (정상 종료가 안 될 때 사용)

### 7.3 시스템 명령어

| 명령어 | 설명 | 사용법 |
|--------|------|--------|
| `uname` | OS 정보 출력 | `uname`, `uname -a` |
| `df` | 디스크 사용량 확인 | `df`, `df -h` |
| `free` | 메모리 사용량 확인 | `free`, `free -h` |

- `-h` 옵션 — KB/MB/GB 등 사람이 읽기 쉬운 단위로 출력
